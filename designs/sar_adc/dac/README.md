# Hybrid DAC - GF180MCU

This directory contains the design, schematics, symbols, and simulation testbenches for the DAC of the **On-Chip SAR ADC**. The DAC is implemented using a **hybrid architecture**, where a resistive array generates scaled reference voltages to reduce the number of capacitors required by the capacitive DAC. The switching network is implemented using a basic 4-to-1 multiplexer, with each switch controlled by the SAR logic output signals `SW0<8:0>` and `SW1<8:0>`.

## Simulation Results

Summary of the DAC performance across PVT corners.

| Parameter | Min | Typical | Max |
|:---|:---:|:---:|:---:|
| Reference Differential Voltage, $V_{RPN}$ | — | 0.90 V | — |
| RDAC Static Power, $P_{RDAC}$ | 23.13 µW | 30.51 µW | 41.49 µW |
| Settling Time (C0 Switching), $t_{s0}$ | 1.417 ns | 1.859 ns | 2.521 ns |
| $t_{s0}/t_{clk}$ | 8.5 % | 11.15 % | 15.13 % |
| Settling Time (C1 Switching), $t_{s1}$ | 1.498 ns | 1.959 ns | 2.654 ns |
| $t_{s1}/t_{clk}$ | ~9 % | 11.75 % | 15.92 % |

### Notes

* The reference differential voltage is defined as $V_{RPN}=V_{RP}-V_{RN}=2.1\mathrm{V}-1.2\mathrm{V}=0.9\mathrm{V}$.
* The reported $P_{RDAC}$ corresponds to the **static power consumption** of the resistive DAC, which is the primary trade-off of the hybrid DAC architecture.
* The settling times $t_{s0}$ and \(t_{s1}\) correspond to switching the least significant capacitors **C0** and **C1**, respectively. These capacitors are connected to the scaled reference voltages generated by the RDAC, making them the worst-case settling scenarios for the DAC.
* The normalized settling times ($t_s/t_{clk}$) are reported with respect to the comparison clock period.
* The slowest settling times were observed under the **SS** process corner with **low $V_{DD}$** and **low temperature**, while the fastest settling times occurred under the opposite conditions (**FF**, **high $V_{DD}$**, and **high temperature**).

## Directory Details

### `cdac/`
* Contains the layout GDS file under `gds/` and the schematic/symbol under `xschem/`.
* Implements the **Capacitive DAC (CDAC)** used for charge redistribution in SAR-based architectures.

### `rdac/`
* Contains the layout GDS file under `gds/` and the schematic/symbol under `xschem/`.
* Implements the **Resistive DAC (RDAC)** as a resistor array for generating discrete voltage levels.

### `dac_sw_1state/`
Provides the transmission gate used for bottom-plate sampling.

* **`dac_sw_1state.sch`**: Xschem schematic of the transmission gate switch.
* **`tb_dac_sw_1state.sch`**: Xschem testbench to evaluate the switch operating as a sample-and-hold circuit.
* **`tb_dac_sw_1state.ipynb`**: Jupyter Notebook that automates transient simulations and post-processes the sampled output to evaluate spectral performance metrics, including **SNDR**, **SFDR**, and **ENOB**.

### `dac_sw_4state/`
Provides the four-state switching network used for passive single-ended-to-differential conversion.

* **`dac_sw_4state.sch`**: Xschem schematic of the four-state switch.
* **`tb_passiveSDC.sch`**: Xschem testbench used to characterize the passive single-ended-to-differential converter and measure its voltage gain.
* **`tb_passiveSDC.ipynb`**: Jupyter Notebook that illustrates the operation of the passive single-ended-to-differential conversion by visualizing the transient waveforms.
* **`tb_setting.sch`**: Xschem testbench used to evaluate the DAC settling time.

## How to Simulate?

### Prerequisites
Make sure you have the following tools installed and configured in your environment:
* **Xschem** (schematic editor)
* **Ngspice** (SPICE simulator)
* **JupyterLab / Jupyter Notebook** (with Python 3 and the required Python packages)

### Running Schematics
1. Launch Xschem from the main repository directory:
   ```bash
   xschem
   ```
2. Open the corresponding `.sch` file to view or simulate the desired testbench.

### Running Jupyter Notebooks
1. In Xschem, generate the SPICE netlist for the corresponding testbench schematic.
2. Launch Jupyter:
   ```bash
   jupyter lab
   ```
   (Or use VS Code)
3. Open the corresponding Jupyter notebook and execute all cells. Each notebook includes a **Simulation setup and run** section where you can configure the main simulation parameters (e.g., VDD, temperature, and process corners).
